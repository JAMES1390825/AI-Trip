# 规划页默认态设计

- 状态：Approved for downstream design

## 1. 背景

当前规划页虽然已经具备：

1. 目的地输入
2. 日期与天数输入
3. 风格与补充要求
4. AI 规划主链调用

但产品感仍然不稳定，主要问题集中在：

1. 画面容易像功能演示稿，而不是成熟 App
2. 卡片和提示过多时，会削弱地图作为主背景的存在感
3. 表单区如果信息层级过重，会让页面更像工具面板而不是旅行内容产品

因此这一轮不是继续加更多字段，而是先锁定：

`规划页默认态应该长成什么样`

## 2. 产品定位

规划页应采用：

`旅行内容产品`

而不是：

1. 强控制台式工具
2. 礼宾服务式高压引导

这意味着页面应优先提供：

1. 轻松开始的感觉
2. 地图与目的地氛围
3. 低门槛输入
4. 少而关键的默认信息

## 3. 页面总结构

### 3.1 背景层

背景直接使用：

`原生接入地图`

不再额外叠加：

1. 旅行氛围大标题
2. 目的地描述性文案
3. 营销式 Hero copy

地图本身就是页面背景和情绪来源。

### 3.2 主交互层

主交互层采用：

`默认中等高度的底部弹层`

要求：

1. 首屏就能看到关键输入
2. 地图仍然保有足够可见面积
3. 不像全屏表单，也不只是一个过小的抽屉

## 4. 默认态内容

### 4.1 首屏可见项

底部弹层默认态首屏应可见：

1. 目的地输入
2. 日期卡
3. 风格标签
4. `开始生成` 按钮

`补充要求（可选）` 默认收起，不应在默认态占据主要视觉位置。

### 4.2 按钮文案

主按钮固定为：

`开始生成`

不使用：

1. `开始生成旅行草案`
2. `开始创建旅程`
3. 其他偏概念化或偏文案化说法

按钮语义应直接、自然、低负担。

## 5. 输入与交互规则

### 5.1 必填项

点击 `开始生成` 前，强制必填只有：

1. 目的地
2. 日期

天数通过：

`开始日期 + 结束日期`

自动确定，不再作为独立必填逻辑存在。

### 5.2 风格标签

风格标签保留在首屏，但它们的角色是：

`软选择`

即：

1. 用户点一下即可直接选中
2. 选中后可随时取消或更改
3. 不把它做成强绑定条件
4. 不要求用户必须先选风格才能生成

### 5.3 目的地输入

目的地输入方式采用：

`在当前底部弹层中直接展开搜索`

不跳转独立搜索页。

原因：

1. 可减少页面跳转感
2. 保持地图和表单在同一上下文
3. 更符合旅行内容产品的轻量输入体验

### 5.4 日期输入

日期与天数组合为：

`一个合并卡片`

点击后在当前底部弹层内部直接展开日历。

用户通过：

1. 选择开始日期
2. 选择结束日期

完成日期与天数的共同确定。

不再把：

1. 日期
2. 天数

拆成两个完全独立的主入口。

### 5.5 补充要求

`补充要求（可选）` 默认收起，只保留轻量入口。

原因：

1. 首屏应优先服务关键输入
2. 可选描述不应压过主要信息
3. 需要时再展开即可

## 6. 缺失项反馈

当用户在未补齐必填项时点击 `开始生成`，反馈方式采用：

`顶部轻提示 + 自动滚到缺项`

而不是：

1. 重弹窗
2. 强干预报错
3. 显性 AI 缺失项卡片

这意味着页面默认态不展示：

`AI 缺失项`

这类系统化提示块。

缺失反馈应该存在，但必须足够轻。

## 7. 视觉原则

### 7.1 地图与弹层关系

地图是主背景，弹层是操作面。

页面气质上应避免：

1. 地图只是装饰图
2. 弹层像独立白底页面压在地图上

正确关系应是：

1. 地图可感知、可作为场景背景
2. 弹层可操作、但不抢走全部注意力

### 7.2 内容层级

默认态层级优先级应为：

1. 地图背景
2. 目的地与日期输入
3. 风格标签
4. 主按钮
5. 可选补充要求

而不是：

1. 系统提示
2. 长文案说明
3. 大段 Hero copy

### 7.3 文案密度

默认态文案应尽量少。

保留：

1. 短标题
2. 简短辅助说明
3. 明确按钮文案

删除：

1. 旅行想象式大文案
2. 对地图背景的解释性文案
3. 冗长 AI 提示语

## 8. 成功标准

该默认态设计成立，意味着：

1. 用户第一次进入时，会先看到真实地图与清晰的底部弹层结构
2. 用户不需要阅读大段说明，就知道要填什么
3. 风格标签存在，但不构成操作压力
4. 目的地和日期都在当前弹层内完成，不打断页面连续性
5. 缺失项反馈存在但足够轻，不破坏页面整体气质

## 9. 后续衔接

这份设计定义只解决：

`规划页默认态`

后续还应继续细化：

1. 目的地搜索展开态
2. 日历展开态
3. 风格标签选中态
4. 开始生成中的过渡态
5. 缺失项轻提示样式

但这些都应服从这份默认态定义，而不是重新改写页面基调。
