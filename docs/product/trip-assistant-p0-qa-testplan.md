更新时间：2026-03-21
负责人建议：QA / 测试工程组

---

## 1. 测试目标

验证 P0 七个模块在“正确性、稳定性、可恢复性、移动端可用性”四个维度达标，并满足 Step 2 指标闸门。

---

## 2. 测试范围

- 多方案生成与对比
- 出发前清单
- 今天执行视图
- 动态重排统一交互
- 版本差异与回退确认
- 只读分享
- 地图与风险诊断

不在 P0 本轮：

- 评论协作
- 外部实时交通服务
- 跨端实时同步

## 5. E2E 主链路（必须通过）

### 链路 E2E-01：从规划到执行

1. 对话补全需求
2. 生成多方案
3. 选择方案并进入 `/trip`
4. 保存行程
5. 切到今天模式
6. 做一次窗口重排
7. 查看差异并回退
8. 生成只读分享链接
9. 访客打开只读页

预期：全链路无阻断，核心埋点落库。

### 链路 E2E-02：异常恢复链路

1. 清空 Amap Key
2. 打开 `/trip`
3. 触发地图诊断
4. 跳转设置并配置 Key
5. 返回重试

预期：用户可在同一会话恢复地图可用。

## 6. 性能与稳定性测试

### 接口性能阈值

- `POST /plans/replan` P95 < 800ms
- `GET /plans/saved/{id}/diff` P95 < 500ms
- `POST /plans/generate`（双方案）P95 < 1.5s

### 前端性能阈值

- `/trip` 首屏可交互 P95 < 1.5s
- 关键交互（时间线点击定位地图）响应 < 200ms

### 稳定性检查

- 24 小时压测无内存异常增长（后端）
- 前端致命错误会话占比 < 0.2%

## 7. 发布前检查清单

- P0 必测用例 100% 通过。
- 阻断级缺陷（S1）= 0。
- 高优先缺陷（S2）关闭率 >= 95%。
- 指标埋点事件字段完整。
- 回滚预案已验证（可回退到上一可用版本）。

---

## 8. 缺陷分级规则

| 级别 | 定义 | 示例 |
|---|---|---|
| S1 阻断 | 主链路无法继续 | 生成后无法进入 `/trip` |
| S2 严重 | 关键功能错误但有临时绕行 | 回退版本号错误 |
| S3 中等 | 体验受损但功能可完成 | 文案错误、状态刷新不及时 |
| S4 轻微 | 视觉或非关键问题 | 对齐、间距、次要提示语 |
